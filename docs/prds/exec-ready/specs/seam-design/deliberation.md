## Investigation Summary

Resolved depth: **lite**. Single whole-piece cluster — the decision unit is the **FR-024 residue**: hardening the seam / integration-coverage contract so a piece that touches an integration boundary cannot silently ship without a behavioral integration AC. Two mechanisms were on the table going in: **Mechanism A** (a seam production-call-site obligation — an author-supplied `src/` pointer naming the production call site each integration AC exercises) and **Mechanism B** (silent-omission closure — a gate that fails a piece which touches a boundary but declares no integration AC).

Key finding from investigation (grounding: research.md; the shipped review-board-integration proposal lineage at `proposals/review-board-integration/spec.md:262-263` + `plan.md:36-37`; behavior-classification.md L51/57; qa-spec criteria #13 and #17; charter NN-C-002/NN-C-003/CR-009): **most of what the two mechanisms proposed already exists in shipped form.** qa-spec **#17** already implements a three-state predicate keyed on `piece_class` (legacy-skip via field-absence / non-behavioral-exempt-with-rationale / behavior-bearing-enforce), and for behavior-bearing pieces it already forces an `[outcome:integration]` AC or an N/A sentinel. qa-spec **#13** already owns integration-AC allocation *including* "any integration silently deferred → must-fix." The shipped integration proposal **already decided** the integration pointer lives on the AC-matrix test-pointer / `Independent Test [machine:]` sub-line, NOT on the facet tag. And review-board-integration's path-coverage axis builds its inventory by tracing the **diff** (`<caller>→<component>`), not by reading any author-supplied pointer.

The genuine residue is therefore narrow and additive, not a new primitive: (1) #17's N/A sentinel is **free text** — a piece can write `Outcome N/A [outcome:integration]: no externals` while actually touching a boundary (the FO-16 failure case); (2) the spec must say honestly which integration failure classes this closes (class 3 untested-unused-seam at spec time) and which stay owned elsewhere (classes 1/2/4/5). No web research fired: every decision unit is a codebase-internal contract question answerable from the charter, the shipped integration lineage, and the qa-spec/behavior-classification docs.

## Integration Check

Single whole-piece cluster → Phase C was a no-op; this section records single-cluster coherence. The surviving paths compose into one additive contract change with no internal conflict:

- **2a (test-sub-line pointer)** and **the behavior-classification.md predicate** are orthogonal by construction: the predicate decides *which facet applies* (facet-applicability, the doc's existing job); 2a carries *where the production call site is* (locator, on the test sub-line). Keeping the locator off the facet tag is precisely what preserves the exactly-one-tag invariant — so the architecture-integrity fix and the scope fix reinforce rather than fight.
- **B-via-#17 (tighten N/A sentinel)** and **B-via-#13 (extend silently-deferred clause)** are two clauses of the same omission-closure concern, both landing on already-shipped criteria — no net-new criterion, so no fragmentation and no retro-fail surface beyond what #13/#17 already impose.
- **5a (front-matter exemption)** rides the same `piece_class` / `behavior_rationale` front-matter machinery #17 already reads — one schema, no new field family.
- **Authoring-path coherence obligation:** because NN-C-003 keys legacy-skip on field-absence, the spec template + spec skill MUST always emit `piece_class` (and any new field this piece adds) for every new spec — otherwise a new author who omits the field is indistinguishable from legacy and silently bypasses the predicate. This is the one cross-touch obligation the surviving set imposes; it is shared with the shipped #17 precedent.
- No cross-unit conflict requiring a VOQ arises from composition. The two surviving operator decisions (below) are about *whether/where to ship*, not about path conflicts.

## Recommendation

Ship the FR-024 residue as a **narrow, additive hardening of the already-shipped integration-coverage contract** — NOT a new "seam-design primitive." Folding all three CONTESTED verdicts, the recommendation lands as:

1. **Pointer placement (architecture Flaw 1 + risk Flaw 1):** the `src/` production-call-site pointer goes on the **`Independent Test [machine:]` sub-line** (path 2a's carrier), NOT on the `[outcome:integration]` facet tag. **REJECT path 1a** — it overloads a negative-space facet with a positive locator and contradicts the shipped integration ADR. The gate is a deterministic SHAPE check (present + src-rooted + not-under-`tests/`).
2. **Omission closure (scope Flaw 1 + architecture Flaw 2):** do NOT mint qa-spec #18. Instead **TIGHTEN the existing qa-spec #17 N/A-sentinel** (reject the free-text `Outcome N/A` when the boundary-touching predicate fires — the FO-16 hole) AND **extend qa-spec #13's existing "integration silently deferred → must-fix" clause** with the boundary-touching predicate. One concern, the two criteria that already own it.
3. **Boundary-touching predicate (architecture Flaw 3 + risk Flaw 2):** define it in **behavior-classification.md**, scoped to facet-applicability ONLY (no locator semantics). State explicitly that it is **JUDGMENT-backstopped (qa-spec judgment), NOT deterministic closure** — an author can dodge by mis-declaring; the spec must not claim deterministic closure.
4. **Exemption rationale (scope Flaw 3):** **5a front-matter** (mirroring `behavior_rationale`). Collapse now; do not ship 5b.
5. **Authoring-path hardening (risk Flaw 3):** the **spec template + spec skill MUST always emit `piece_class`** (and any new field this piece introduces) for every new spec, so a new author who omits the field is not mistaken for legacy (NN-C-003 field-absence-skip). Without this, the predicate is bypassable for every new spec.
6. **Honest scope statement (scope Flaw 2):** the spec MUST state that this closes **integration failure class (3) untested-unused-seam at spec time**, weakly assists class (5) contract-drift, and does NOTHING for classes (1) mock-avalanche, (2) cross-phase composition, (4) post-merge conflict — those remain owned by the shipped review-board-integration path-coverage axis and the plan registry. Do not name or frame the piece as a general "seam primitive."

NN-C-002 (markdown/bash/yaml only — no AST tooling, which is why 2c is out), NN-C-003 (additive only), CR-009, and NN-C-009 (version bump + CHANGELOG) constrain the build.
